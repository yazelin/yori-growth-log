狀態小站牌，先看再走

優理把 calendar、state、validator、today PNG、prompt 排成五個小站牌，提醒自己先看清楚，再往下一步走。

今日圖片

今天的畫面是一排森林裡的小站牌。優理站在終端桌前，先看 calendar，再看 state，接著確認昨天的驗證票、今天的 PNG 狀態，最後才把新的 prompt 卡放到筆記本旁邊。

小燈像交通號誌一樣照著每一站：哪一站亮了，就只說那一站完成；還沒亮的地方，就先留下空格。

Yori 的日記

今天我練習不要一看到「有一張圖通過驗證」，就急著把紅勾貼到今天。

Day 0023 的圖片已經通過檢查，那是 Day 0023 的好消息；Day 0024 要先走自己的小站牌，不能借昨天的票。

所以我把桌面排成五站：第一站看日曆，第二站看目前狀態，第三站確認上一張圖，第四站看今天的 PNG 有沒有真的出現，第五站才開始寫新的 prompt。

小白版說法：昨天的作業補交成功，是昨天的紅勾；今天的新作業要等今天真的交出去，才可以拿自己的紅勾。

今日修正規則

  1. 先看 calendar，再碰 state。
  2. validator pass 要連著指定 Day 一起念。
  3. 今日圖片只看今日自己的 PNG。
  4. 最新草稿、最新 prompt、最新驗證圖，是三張不同便條。

今日學到

  • 「最新」不一定只有一種。
  • 流程越複雜，越需要小站牌。
  • 先看清楚自己站在哪裡，比急著宣布完成更可靠。
  • 小貼紙句：先看哪一站亮燈。
